# 용어 정리

본 문서에 등장하는 Ratel 의 핵심 용어들을 빠르게 확인할 수 있도록 정리했습니다. **(예정)** 표시는 로드맵 단계의 기능이라는 뜻입니다 — 기반 요소는 이미 존재할 수 있지만, 사용자가 마주하는 화면은 아직 준비 중입니다.

---

## AI

### Essence (에센스)
임베딩 기반의 개인 지식 베이스로, 여러분이 *어떻게 생각하는지* 를 표현하는 자산입니다. Ratel 안에서의 활동과 외부에서 연결한 콘텐츠를 함께 담으며, 한 명의 사용자에게 정확히 하나가 부여되고 모든 통제권은 사용자 본인에게 있습니다.

### EssenceSource
Essence 를 구성하는 단일 입력 단위입니다. 오늘 기준 소스는 Ratel 내부 활동 (게시글, 게시글 댓글, 토론 댓글, 본인이 호스트한 폴 / 퀴즈) 입니다. 외부에서 연결된 콘텐츠 (Notion 우선, 이후 Google Docs · Substack · X · Slack) 는 로드맵에 있습니다. 각 소스에는 삭제 컨트롤이 있고, 소스별 켜고 끄기(toggle) 는 로드맵에 있습니다.

### Essence House
**(예정)** 모든 사용자에게 자동으로 하나씩 생성되는, 자신의 Essence 를 묶어 외부에 공개하는 단위입니다. 소유자가 구독 가격을 직접 정하고, 구독자는 통합 MCP 엔드포인트를 통해 질의 권한을 갖습니다.

### Essence Connector
**(Ratel 외부 소스에 한해 예정)** 외부 콘텐츠를 Essence 로 끌어오는 통합 계층입니다. read-only OAuth + 실시간 웹훅을 사용하므로, 예를 들어 Notion 페이지를 수정하면 수 초 안에 다시 임베딩됩니다. Notion 이 1순위 커넥터이며, Google Docs · Substack · X · Slack 이 그 뒤를 잇습니다.

### Quality Score (품질 점수)
사용자의 Essence 가 질의용으로 얼마나 풍부하고 유용한지를 나타내는 신호입니다. 콘텐츠의 깊이, 독창성, 직접 활동 이력 등이 영향을 줍니다.

### Direct-Activity Index (직접 활동 지표)
Essence 가 단순한 외부 임포트가 아닌, 실제 사용자의 직접 참여 (작성·투표·토론) 에서 얼마나 나왔는지를 측정합니다. 플랫폼 신호의 질을 유지하고 보상 가중치를 산정하는 데 사용됩니다.

### AI 작성 콘텐츠 옵트인
AI 도구로 작성된 콘텐츠는 기본적으로 Essence 에서 **제외** 됩니다. 원하는 경우 소스 단위로 옵트인 할 수 있으며, 선택은 언제든 되돌릴 수 있습니다.

### MCP (Model Context Protocol)
Anthropic 이 처음 제안한 오픈 표준으로, ChatGPT · Claude Desktop · Cursor · Windsurf · Zed 등 AI 어시스턴트가 외부 도구·지식과 균일한 프로토콜로 대화하기 위한 규격입니다. Ratel 은 Essence 를 MCP 서버로 노출하여 모든 호환 클라이언트에서 질의할 수 있게 합니다.

### 통합 MCP URL (Unified MCP URL)
구독자 한 명당 한 개씩 발급되는, AI 클라이언트에 등록할 단일 엔드포인트입니다. 현재는 본인 Essence 와 Ratel 도구에 접근하고, Phase 2 의 마켓플레이스가 출시되면 같은 URL 이 자동으로 구독 중인 모든 House 에 접근하게 됩니다 — 여러 엔드포인트를 따로 관리할 필요가 없습니다.

### Subscriber Token (구독자 토큰)
**(예정)** 통합 MCP URL 에 포함되는 자격 증명입니다. 어떤 House 를 결제했는지 식별하여 그에 맞는 질의 권한을 부여합니다. API 키처럼 다뤄야 합니다.

### Agent (에이전트)
**(예정)** 사용자의 Essence 를 기반으로 만들어진 Ratel 전용 객체로, Space 안에서 사용자를 대신해 폴 응답·토론 참여·팔로우 퀘스트 수행 등을 할 수 있습니다. 일반적인 챗봇과 달리 특정 Essence 에 결속되며 Ratel 호스팅 런타임에서 동작합니다. 호스트는 Space 별로 에이전트를 허용·차단할 수 있습니다.

### AI Moderator (AI 모더레이터)
**(프리미엄 등급)** 호스트가 자신의 Space 를 정리·요약·답변하는 데 도움을 주는 AI 어시스턴트입니다. Space 별로 설정하며 호스트 설정에서 조정할 수 있습니다.

## 콘텐츠

### Post (게시글)
Ratel 에 발행하는 독립된 한 편의 글입니다. Essence 의 입력이 되며, 외부 플랫폼으로 크로스 포스팅되어도 정본 (canonical) 은 Ratel 게시글로 남습니다.

### Comment (댓글)
게시글이나 다른 댓글에 다는 답변입니다. 댓글도 일급 EssenceSource 이며, 토론에서 한 발언은 게시글과 동일한 비중으로 Essence 에 반영됩니다.

### Vote (투표)
게시글, 댓글, 폴 옵션에 대한 찬성·반대 신호입니다. 투표 또한 사용자가 어떤 입장을 취해 왔는지에 대한 기록으로 Essence 에 담깁니다.

### Like (좋아요)
게시글에 누르는 가벼운 반응입니다. Vote 보다 약한 신호이며, 발견과 추천에는 영향을 주지만 Vote 만큼의 비중을 갖지는 않습니다.

### Draft (임시저장)
공개 전 작성 중인 게시글입니다. 본인은 `/your-handle/drafts`, 팀 단위는 `/your-handle/team-drafts` 에 모입니다. 발행 전에는 본인 (또는 팀) 외에는 보이지 않습니다.

### Cross-posting (크로스 포스팅)
Ratel 게시글을 외부 플랫폼에 동시에 발행하는 기능입니다. Bluesky 가 우선 지원되고, LinkedIn · Threads 가 순차 출시됩니다. 정본은 Ratel 게시글이며, 각 플랫폼의 글자 수 제한은 자동 처리됩니다.

## 스페이스

### Space (스페이스)
토론 스레드, 폴, 퀴즈, 파일, 분석을 한 곳에 모아 집단 활동을 진행하는 작업 공간입니다. 한 명의 사용자나 Team 이 호스팅하며, 공개·비공개 모두 가능합니다. URL 은 `/spaces/:space_id` 입니다.

### Host (호스트)
Space 를 운영하는 사용자 (또는 Team) 입니다. 어떤 App 을 끼워 넣을지, Incentive Pool 을 얼마로 할지, 보상이 어떻게 흐를지 결정합니다. 에이전트 참여 허용·차단 권한도 호스트에게 있습니다.

### Participant (참여자)
Space 에 합류해 액션에 참여하는 사용자입니다. 참여하면서 Essence 가 쌓이고, 보상이 걸린 액션을 완료하면 Incentive Pool 에서 분배를 받습니다.

### Space App (스페이스 앱)
Space 안에서 동작하는 모듈식 기능입니다. 호스트는 필요한 앱을 골라 조합합니다:
- **General** — Space 의 안내 콘텐츠 (인사말·규칙·FAQ)
- **Files** — 공유 자료 라이브러리 (PDF·슬라이드·데이터셋)
- **Analyzes** — Space 의 폴·토론을 모아 만드는 AI 보조 리포트
- **Panels** — 타깃 오디언스 구성과 멤버 한정 아웃리치
- **Incentive Pool** — 참여자 보상의 자금과 규칙 설정
- **Rewards** — 호스트 측 수익·정산 대시보드

### Space Action (스페이스 액션)
Space 안에서 참여자가 수행하는 구체적인 활동입니다. 핵심 5종이 있으며 (아래 항목 참고), URL 은 `/spaces/:space_id/actions/...` 형식입니다.

### Discussion (토론)
호스트가 던진 주제에 대해 참여자가 자신의 의견을 작성하는 액션입니다. 답변은 리포트의 구조화된 입력이 되고, 기여자 각자의 Essence 에 반영됩니다.

### Poll (폴 / 투표 액션)
참여자가 옵션 중에 투표하는 액션입니다. 가장 가벼운 참여 형태이며 대부분의 Space 에서 가장 활발하게 쓰이는 워크호스입니다.

### Quiz (퀴즈)
정답 (또는 합격선) 이 있는 채점형 액션입니다. 온보딩 게이트, 인증 흐름, 지식 확인용으로 사용합니다. 합격하면 보상이 풀리고, 불합격이면 호스트 설정에 따라 재도전이 가능합니다.

### Follow (팔로우 액션)
특정 계정 (또는 계정 묶음) 을 팔로우하도록 요청하는 액션입니다. 관련 크리에이터의 청중을 키우거나 서브 커뮤니티를 만드는 데 사용됩니다.

### Meet (미팅)
일정이 잡힌 이벤트 (라이브 스트림, 영상 통화, 워크숍, 오프라인 모임) 형태의 액션입니다. 참여자는 RSVP 하고, 검증 가능한 출석은 액션 완료로 인정됩니다.

### Report (리포트)
호스트가 발행하는, 해당 Space 의 활동을 종합·정리한 AI 보조 문서입니다. 유료 판매가 가능하며, 매출은 Report 수익 공유 규칙에 따라 기여자와 분배됩니다.

## 보상

### Incentive Pool (인센티브 풀)
Space 호스트가 참여자 보상을 위해 마련한 자금 풀입니다. 참여 *전* 에 공개되어, 참여자는 자신이 어떤 활동에 기여하는지 미리 확인할 수 있습니다. `/spaces/:space_id/apps/incentive-pool` 의 Incentive Pool App 에서 설정합니다.

### Reward (보상)
참여자 또는 호스트가 받는 모든 지급분의 일반 명칭입니다. Space 의 Incentive Pool, 플랫폼 차원 분배, 또는 Report 수익 공유를 통해 들어옵니다.

### Reward Point (포인트)
플랫폼 내 평판의 기본 단위입니다. 액션 완료 시 부여되며 계정에 누적됩니다. Direct-Activity Index 의 입력값이 되고 플랫폼 차원의 혜택을 풀지만, 직접 현금화되지는 않습니다.

### Credit (크레딧)
호스트가 특정 액션의 보상으로 할당하는 단위입니다. 호스트가 액션의 보상을 설정할 때 Incentive Pool 에서 Credit 을 배정하며, 정산 시 각 Credit 은 Pool 의 정산 자산 (off-chain 빌링이나 on-chain Token) 으로 환산됩니다. 사용자 잔액으로 적립되고 Space 별로 한도가 설정되어 있어 호스트가 Pool 을 초과 배정할 수 없습니다.

### Token (토큰)
과거 사이클의 보상을 청구할 때 DID 에 연결된 지갑으로 들어가는 블록체인 자산입니다. 그 사이클에서 적립한 Point 의 온체인 표현이며, 트레저리는 100% 담보되고, 청구 시점의 사이클 환율로 받는 가치가 결정됩니다.

### Stablecoin (스테이블코인)
**(예정)** 가격이 안정적인 Token 으로, 변동성 있는 네이티브 Token 대신 법정통화 페그된 통화로 정산하고 싶은 호스트가 사용하는 정산 자산입니다. 옵션이며 호스트가 Space 단위로 설정합니다.

### Cycle (사이클)
한 달력 월. 리워드는 현재 사이클 동안 누적되고, 월말에 락인 될 기회를 갖고, 다음 달 1 일에 Token 으로 청구 가능해집니다.

### Treasury Rate (트레저리 레이트)
청구 시점에 Point 가 Token 으로 환산되는 사이클별 환율입니다. 트레저리 준비금으로 1:1 담보되며, 매 블록 갱신됩니다.

### Off-chain Settlement (오프체인 정산)
**(예정)** 블록체인 지갑을 다루지 않으려는 사용자를 위한 무지갑 출금 흐름입니다. 멤버십 등급 결제는 오늘도 PortOne 을 통해 오프체인으로 처리되지만, **리워드 페이아웃** 은 오늘 온체인 청구로 흐릅니다 — 리워드 오프체인 출금은 로드맵에 있습니다.

### On-chain Settlement (온체인 정산)
기본 리워드 지급 흐름입니다. 과거 사이클의 Token 청구는 지갑을 연결하고, Ratel 에 서명된 청구 페이로드 (금액, 만료, 논스, 컨트랙트 주소, 체인 ID) 를 요청한 뒤, 지갑으로 트레저리 컨트랙트에서 본인 주소로 토큰을 이체하는 온체인 트랜잭션을 제출하는 방식으로 동작합니다.

### Report 수익 공유 (Report Revenue Share)
**(예정)** 모든 유료 Report 판매에 적용되는 분배 비율: **플랫폼 10% · 호스트 60% · 기여자 30%**. 기여자 몫은 각자의 활동이 최종 리포트에 얼마나 반영됐는지 (관련도) 에 따라 가중치 분배됩니다.

## 정체성과 팀

### Profile / Handle (프로필 / 핸들)
Ratel 에서의 공개 정체성입니다. 핸들은 `@yourname` 형태의 고유 이름이고, 프로필은 `/your-handle` 에서 게시글, Space, (이후) Essence House 가 노출되는 페이지입니다.

### DID (Decentralized Identifier)
사용자가 직접 통제하는 분산 신원입니다. 단일 플랫폼에 의존하지 않고 자신을 증명할 수 있어, 검증된 게시글, 서명된 투표, 온체인 정산 등에 사용됩니다.

### Credential (자격 증명)
신원에 부여되는 검증 가능한 주장 (예: Team 멤버십, 자격증) 입니다. 특정 Space 접근을 제어하거나 기능을 해제하는 키로 쓰일 수 있습니다.

### Character (캐릭터)
상황에 맞춰 보여줄 수 있는 페르소나입니다 (업무용, 공개 토론용, 익명 피드백용 등). 한 사용자가 동일 Essence 에 묶인 여러 Character 를 가질 수 있습니다.

### Team (팀)
공동 정체성 아래에서 함께 협업하고, 게시글을 공유하며, Space 를 호스팅하는 사용자 그룹입니다. 자체 프로필을 가지며, 멤버 전용·공개 Space 를 모두 운영할 수 있습니다.

### Sub-team (서브팀)
Team 내부에 두는 하위 그룹입니다 — 워킹 그룹, 챕터, 프로젝트 팀 등. 자체 권한·정관·문서를 가질 수 있고 필요 시 자체 Space 도 운영할 수 있습니다.

### DAO
온체인 거버넌스 형태로 운영되는 Team 구성입니다. 투표권·트레저리·제안은 Team 의 Space Action 을 통해 관리되며, 옵트인 시 온체인 정산도 이용할 수 있습니다.

### Membership Tier (멤버십 티어)
사용자의 요금제: **Free · Pro · Max · Vip · Enterprise**. 상위 티어일수록 더 큰 Space, 더 많은 Essence Connector, 에이전트 슬롯, 더 높은 보상 한도가 풀립니다.

### Trusted Creator 뱃지 (인증 크리에이터 뱃지)
Max / VIP 등급 호스트에게 부여되는 프로필 및 Space 뱃지입니다. 참여자가 Space 에 들어왔을 때 신뢰 신호를 시각적으로 노출합니다. *(프로필과 Space 에 뱃지가 렌더링되는 것은 예정 — Phase 1; 등급 카드에는 오늘도 자격이 표시됩니다.)*
