# 보상

Ratel 의 보상 경제는 활동을 가치로 바꾸는 다섯 가지 단위로 구성되어 있고, 각각 역할이 다릅니다.

- **XP (경험치)** 는 스페이스 액션을 완료할 때 자동으로 적립되고 캐릭터 레벨을 올립니다.
- **포인트(Reward Points)** 는 인플랫폼 평판 단위 — 보상 스페이스 전반을 가로지르는 *적립의 단위* 입니다.
- **크레딧(Credits)** 은 액션 보상을 뒷받침하는 호스트 펀딩 통화 — 포인트의 상류예요.
- **토큰(Tokens)** 은 사이클이 잠긴 뒤 청구할 때 포인트가 변환되는 온체인 지급 단위입니다.
- **인센티브 풀(Incentive Pool)** 은 호스트가 사전에 마련하는 펀딩 메커니즘 — 액션 보상이 처음 채워지는 곳이에요.

보상은 세 곳에서 흘러나옵니다 — 스페이스의 Incentive Pool (호스트 펀딩), 플랫폼이 분배하는 풀 (게시글 등 네이티브 활동의 참여 가중 적립), 그리고 출시되면 *(예정)* Phase 4 의 유료 리포트 수익 공유 (플랫폼 10% · 호스트 60% · 기여자 30%).

`/<your-handle>/rewards` 를 열면 적립한 것, 진행 중인 것, 청구 가능한 것을 모두 볼 수 있어요. 페이지는 **사이클 (한 달 단위)** 을 중심으로 구성됩니다 — 보상은 현재 사이클에 누적되고, 월말에 잠기고, 다음 달 1 일에 토큰으로 청구 가능해집니다.

## 이 챕터에서

| 페이지 | 다루는 내용 |
|---|---|
| **[경험치 (XP)](./xp.md)** | 활동 기반 성장 단위. 액션 유형별 (Poll, Quiz, Discussion, Follow) 고정 XP 값, XP 가 캐릭터 레벨로 어떻게 묶이는지, 중복 적립을 막는 dedup 규칙. |
| **[포인트](./points.md)** | 인플랫폼 평판 단위 (`pts`). 포인트 적립 방식, 직접 활동 지표(Direct-Activity Index) 에 반영되는 경로, 캐릭터 스킬 트리 부스트가 곱셈으로 늘리는 방식. |
| **[크레딧](./credits.md)** | 액션 보상을 뒷받침하는 호스트 펀딩 통화. Credit 의 출처, 구독 등급별 할당량, 지급 시 포인트로 변환되는 방식. |
| **[토큰](./tokens.md)** | 과거 사이클 포인트의 온체인 표현. 2 단계 청구 흐름 (지갑 연결 → 서명 후 제출), 트레저리 레이트, 100% 담보 모델. |
| **[인센티브](./incentives.md)** | 호스트 측 펀딩 화면 — `/spaces/:space_id/apps/incentive-pool` 에서 인센티브 풀 설정, 분배 규칙 (균등 / 깊이 가중 / 단계별), 그리고 지급 이력을 감사 가능하게 유지하는 방법. |
